The output column says what each row materialises: `numpy frames` are
frame objects holding numpy arrays (oxyz's native form, and cextxyz's via
its `read_dicts` interface), `ase.Atoms` are full Atoms objects (ASE's
own reader, `oxyz.ase.read`, and the C parser behind ASE's IO via the
ase-extxyz plugin), and `Batch` is a single concatenated batch. Rows with
different outputs do different amounts of work. The mode column says
whether parsing uses all cores or one.

Throughput columns divide the work a row does (frames and atoms read,
recorded with the save) by the mean time; rows where a per-frame rate
means nothing (first/last/scan) leave them blank. Rows suffixed `-Nt`
are the same reader pinned to N threads.
"rel" is relative to the fastest row in the group. Numbers are only
comparable within one run on one machine.

## Store comparison

The `stores/...` and `stores-ase/...` groups read the 100k-frame dataset
through database-style stores it was ingested into once, up front. The
stores read pre-converted binary through an index; oxyz parses the text
file and nothing else, so the stores are expected to lead — the margin
is the result. One reason per baseline: atompack is the specialised
binary molecule store (mmap-backed, rayon-parallel, float32 by design);
lmdb-pickle is the LMDB-of-arrays pattern dataloader repositories
hand-roll; ase-sqlite and ase-lmdb are the common ASE ecosystem paths.
`stores/...` rows hand back numpy arrays and `stores-ase/...` rows
ase.Atoms; the two output contracts are never compared in one table.

## Where oxyz is not fastest

Groups where an oxyz row is not the fastest. For store groups this is the expected outcome — binary stores against a text parser — and the margin is the result:

- stores-ase/sequential: `ase-lmdb` leads; oxyz's best row is 2.6× slower
- stores-ase/shuffled: `ase-lmdb` leads; no oxyz row
- stores-ase/strided: `ase-lmdb` leads; oxyz's best row is 39.8× slower
- stores/sequential: `atompack-native` leads; oxyz's best row is 18.3× slower
- stores/shuffled: `atompack-native` leads; oxyz's best row is 146.9× slower
- stores/strided: `atompack-native` leads; oxyz's best row is 137.5× slower
